THUAT TOAN: Phan tich so thanh thua so nguyen to (Prime Factorization)
DO PHUC TAP: O(sqrt(n))

PSEUDO CODE:
function primeFactorization(n):
    factors = []
    
    // Kiem tra uoc so 2
    while n % 2 == 0:
        factors.append(2)
        n = n / 2
    
    // Kiem tra cac uoc so le tu 3 tro di
    i = 3
    while i * i <= n:
        while n % i == 0:
            factors.append(i)
            n = n / i
        i = i + 2
    
    // Neu n > 1 thi n la so nguyen to
    if n > 1:
        factors.append(n)
    
    return factors


THUAT TOAN: Tim so nguyen to gan nhat
DO PHUC TAP: O(n * sqrt(n))

PSEUDO CODE:
function isPrime(n):
    if n < 2:
        return false
    if n == 2:
        return true
    if n % 2 == 0:
        return false
    for i = 3 to sqrt(n) step 2:
        if n % i == 0:
            return false
    return true

function findNearestPrime(target):
    if target <= 2:
        return 2
    
    lower = target - 1
    upper = target + 1
    
    while lower >= 2 or upper < infinity:
        if lower >= 2 and isPrime(lower):
            return lower
        if isPrime(upper):
            return upper
        lower = lower - 1
        upper = upper + 1
    
    return 2

MÔ TẢ BẰNG NGÔN NGỮ TỰ NHIÊN:

1. Phân tích số thành thừa số nguyên tố:
   - Đầu tiên xử lý ước số 2: liên tục chia n cho 2 và ghi lại số lần chia
   - Sau đó xử lý các ước số lẻ: duyệt các số lẻ từ 3 đến căn n
   - Với mỗi ước số tìm được, chia liên tục cho đến khi không chia hết
   - Sau khi chia hết tất cả ước số nhỏ hơn căn n, nếu n vẫn lớn hơn 1 thì n còn lại chính là một thừa số nguyên tố
   - Thu thập tất cả các thừa số nguyên tố tìm được

2. Kiểm tra số nguyên tố:
   - Loại trừ các trường hợp đặc biệt: số nhỏ hơn 2 không phải SNT, 2 là SNT duy nhất chẵn
   - Với các số khác, kiểm tra xem có chia hết cho số nào từ 3 đến căn n không
   - Chỉ cần kiểm tra các số lẻ (bước nhảy 2) vì số chẵn đã loại trừ
   - Nếu không tìm thấy ước số nào, đó là số nguyên tố

3. Tìm số nguyên tố gần nhất:
   - Bắt đầu từ số cần tìm, mở rộng sang hai phía: nhỏ hơn và lớn hơn
   - Tại mỗi bước, kiểm tra số phía dưới (target - 1, target - 2, ...)
   - Đồng thời kiểm tra số phía trên (target + 1, target + 2, ...)
   - Số nguyên tố nào tìm được trước sẽ là số nguyên tố gần nhất
   - Ưu tiên số nhỏ hơn nếu cả hai cách đều xa như nhau
